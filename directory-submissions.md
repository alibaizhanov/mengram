**Long description (200 words):**
> Mengram is an AI memory layer purpose-built for agents and developers. Unlike fact-only memory tools, Mengram stores three memory types: **semantic** (facts and preferences), **episodic** (events, decisions, outcomes), and **procedural** (workflows that evolve from successes and failures).
>
> The procedural layer is the differentiator: when a workflow fails, the revision records the *violated assumption* — the specific belief that turned out false — and derives a precondition that travels with the workflow at recall time. Agents load a proven v3 instead of re-deriving a v1, and know what to verify before trusting it. Most memory tools store only facts; almost none learn how you work.
>
> Mengram ships as a hosted REST API, a Model Context Protocol (MCP) server (listed in the official MCP registry, compatible with Claude Code, Claude Desktop, and Cursor), Python/JavaScript SDKs, and first-party integrations for LangChain (`langchain-mengram`), CrewAI, and Hermes Agent. Multi-user isolation lets you ship memory inside your own agent product, one API key per end-user. Native multilingual retrieval works in 23 languages.
>
